◆ from_first_available()

static Camera Prophesee::Camera::from_first_available ( )
static

Initializes a camera instance from the first available camera plugged on the system.

Open the first available camera following at first EMBEDDED and then USB order.
Please note that remote cameras will not be opened with this function. To do that, please specify the OnlineSourceType and use the Camera::from_source function, or else specify the serial number and use the Camera::from_serial function.
Serial numbers and types of available sources can be found with Camera::list_online_sources function.
Throws CameraException in case of initialization failure.

◆ set_exposure_frame_callback()

void Prophesee::Camera::set_exposure_frame_callback ( std::uint16_t  fps,
ExposureFrameCallback  exposure_frame_callback 
)

Sets the callback that will be called at specified framerate and pass as argument a frame of exposure timings for each pixel.

Note that this frame is calculated by taking into account the difference of time (the true exposure time) between the first and second EM, but also the difference of time with the previous CD event and the polarity of the CD event itself. Thus, this frame is different from the one that would be computed using only the EM events :

  • its pixels are updated more frequently than by using only the EM
  • the exposure time measurements are approximated in between EM by using CD, which is especially useful when the scene is changing rapidly

Note that the frame is updated continuously, so you should make a copy of the frame passed as argument to your callback if you plan on using it later.

Note that only one such callback can be set at anytime.

Parameters
fpsFramerate at which the callback will be called (in number of frames per second)
exposure_frame_callbackCallback to call at required framerate
Examples:
prophesee_raw_to_video.cpp, and prophesee_viewer.cpp.

◆ set_max_event_rate_limit()

bool Prophesee::Camera::set_max_event_rate_limit ( uint32_t  rate_kEV_s)

Sets the maximum event rate in kilo events per seconds to be sent by the camera in the pipeline.

By default, no maximum event rate is set.

For cameras generations >= GEN4 : -> calling set_max_event_rate_limit when replaying a RAW file recorded with a camera gen >= GEN4 is unsupported -> with this feature, you cannot drop more than 31 events over 32, so that if the actual event rate is too elevated, you could get more events than expected (more specifically, you cannot get lower than 1/32 of the actual event rate)

Throws CameraException if the camera has not been initialized.

Parameters
rate_kEV_sMaximum event rate in kilo events per second. The zero value corresponds to no max event rate set.
Returns
True if the limit has been set correctly.
Examples:
prophesee_viewer.cpp.

◆ set_max_events_lifespan()

bool Prophesee::Camera::set_max_events_lifespan ( Prophesee::timestamp  max_events_lifespan_us)

Sets the maximum event lifespan.

Only the events that occurred in the last max_events_lifespan_us microseconds are processed.

By default, this value is set to 1 second (if the feature is supported for the camera's generation). Remark : this feature is not available with cameras from generations >= GEN4

Events occurred earlier are ignored. Throws CameraException if the camera has not been initialized.

Parameters
max_events_lifespan_usMaximum event lifespan in microseconds. This value cannot be larger than 1 second.
Returns
True if the parameter has been set correctly.

◆ synchronize_and_start_cameras()

static bool Prophesee::Camera::synchronize_and_start_cameras ( Camera &  master,
Camera &  slave 
)
static

Synchronizes and starts two prophesee cameras.

Use this function to configure the behavior of the cameras only in the context of a camera synchronization (i.e. multicamera setup). The correct pins of the master and slave cameras must be linked. The synchronization is basically made by the master camera: it sends a signal to the slave(s) camera each time its clock updates.

Warning
This is supported by GEN3 EVK cameras only.
This is supported only with live cameras (no offline source).
Parameters
masterThe master camera
slaveThe slave camera
Returns
True if both cameras started successfully, false otherwise. Also returns false, if one of the camera was already started.

Throws CameraException if the feature is not supported.

Example :

Prophesee::Camera camera_master = Prophesee::Camera::from_first_available(); // or Prophesee::Camera::from_serial(master_serial) to specify which camera is master Prophesee::Camera camera_slave = Prophesee::Camera::from_first_available(); // or Prophesee::Camera::from_serial(slave_serial) to specify which camera is slave

camera_slave.cd().add_callback([&](const Prophesee::EventCD *ev_begin, const Prophesee::EventCD *ev_end) { // your code here }); camera_master.cd().add_callback([&](const Prophesee::EventCD *ev_begin, const Prophesee::EventCD *ev_end) { // your code here }); Prophesee::Camera::synchronize_and_start_cameras(camera_master, camera_slave);

while (camera_master.is_running()) { // your code here } camera_slave.stop(); camera_master.stop();
