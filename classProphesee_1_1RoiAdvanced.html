Detailed Description

Facility class to handle advanced use of the sensor ROIs.

The hardware ROI is equivalent to an & operator on the sensor matrix. When setting a ROI you enable some lines and some columns of the sensor. A pixel is then enabled if and only if the line and column it lays on are enabled.

void Prophesee::RoiAdvanced::set ( const std::vector< Roi::Rectangle > &  to_set)

Sets several rectangular hardware ROI from Roi::Rectangle vector.

Any line or column enabled by a single ROI is also enabled for all the other (see RoiAdvanced description). Example: Input vector is composed with 2 rois (0, 0, 50, 50), (100, 100, 50, 50). In the sensor, it will result in 4 regions as follow

  • (0, 0, 50, 50)
  • (0, 100, 50, 50)
  • (100, 0, 50, 50)
  • (100, 100, 50, 50) Indeed, rows and columns from 0 to 50 and 100 to 150 are enabled.

Example: Input vector is composed with 2 ROIs: (0, 0, 50, 50), (25, 25, 50, 50). In the sensor, it will result in 1 region as follow

  • (0, 0, 75, 75)
